# Типографика

> Figma: TBD

Типографика SEDA UI задаёт текстовую иерархию интерфейса независимо от размера компонента. Размеры компонентов описаны в `sizes.md`; этот документ описывает роли текста: заголовки, основной текст, подписи, вспомогательный текст, метрики и код.

---

## Цель

Типографика должна помогать пользователю быстро понять:

- где главный смысл экрана;
- какие блоки связаны между собой;
- что является действием, подписью, метрикой или вспомогательной информацией;
- какие данные нужно сравнить или просканировать.

SEDA UI ориентирована на плотные B2B и AI-интерфейсы, поэтому типографика должна быть спокойной, читаемой и предсказуемой, без декоративной конкуренции с данными.

---

## Стек Шрифтов

| Токен | Значение | Назначение |
|---|---|---|
| `font/family` | `SF Pro Text` | Основной шрифт в текущих токенах |
| `font/family/fallback` | `Inter`, `Segoe UI`, `Roboto`, `Arial`, `sans-serif` | Запасной стек для Web |
| `font/family/mono` | `SFMono-Regular`, `Consolas`, `Liberation Mono`, `monospace` | Код, идентификаторы, технические значения |

Рекомендуемый CSS stack:

```css
font-family: "SF Pro Text", Inter, "Segoe UI", Roboto, Arial, sans-serif;
```

---

## Модель Типографических Токенов

Текущие foundation-токены уже содержат размер, line-height, weight и letter-spacing. Каждый текстовый токен имеет варианты:

| Вариант | Насыщенность | Когда использовать |
|---|---|---|
| `base` | Regular | Обычный текст |
| `strong` | Semibold / Medium | Акцент, заголовок, важное значение |
| `emphasis` | Italic | Редкий смысловой акцент, цитаты, редакторские примечания |

Не используйте `emphasis` как декоративный стиль для обычных подписей интерфейса.

## Базовая Шкала Типографики

| Токен | Базовый размер / строка | Strong | Emphasis | Основное назначение |
|---|---:|---|---|---|
| `font/titleLarge1` | 48 / 64 | Semibold | Light Italic | Очень крупный презентационный заголовок, редкие hero-сценарии |
| `font/titleLarge2` | 40 / 56 | Semibold | Italic | Презентационные заголовки |
| `font/titleLarge3` | 32 / 48 | Semibold | Italic | Крупные заголовки страниц |
| `font/title1` | 24 / 40 | Semibold | Italic | Заголовок страницы или крупной секции |
| `font/title2` | 20 / 32 | Semibold | Italic | Заголовок секции |
| `font/title3` | 18 / 24 | Semibold | Italic | Заголовок панели, карточки или группы |
| `font/headline` | 16 / 20 | Semibold | Italic | Компактный заголовок, подпись заголовка таблицы или панели |
| `font/bodyExtraLarge` | 18 / 28 | Semibold | Italic | Крупный основной текст или текст онбординга |
| `font/bodyLarge` | 16 / 24 | Semibold | Italic | Основной текст повышенной читаемости |
| `font/body` | 14 / 20 | Semibold | Italic | Базовый текст интерфейса |
| `font/bodySmall` | 12 / 16 | Medium | Medium Italic | Компактный интерфейс, ячейки таблиц, вторичные подписи |
| `font/bodyExtraSmall` | 10 / 14 | Semibold | Italic | Микротекст, плотные метаданные |
| `font/caption` | 8 / 10 | Semibold | Italic | Очень редкие технические подписи |

> Примечание о текущем состоянии: `titleLarge1`, `titleLarge2`, `titleLarge3` и часть `title1` в `tokens.json` используют отрицательный letter-spacing. Для production UI используйте только значения из токена и не добавляйте отрицательный letter-spacing вручную. Для плотных интерфейсов предпочтительно держать letter-spacing `0`.

## Семантические Роли

Семантические роли описывают назначение текста. Они мапятся на базовые токены, но не заменяют их.

### Презентационный Текст

| Роль | Токен | Использование |
|---|---|---|
| `type/display/large` | `font/titleLarge1/strong` | Редкий hero или презентационный экран |
| `type/display/medium` | `font/titleLarge2/strong` | Крупная промо-страница, onboarding |
| `type/display/small` | `font/titleLarge3/strong` | Крупный landing/page intro |

Не использовать:
- В плотных дашбордах.
- В таблицах, формах, сайдбарах, модальных окнах.
- Для обычных заголовков секций.

### Заголовки

| Роль | Токен | Использование |
|---|---|---|
| `type/heading/page` | `font/title1/strong` | Главный заголовок страницы |
| `type/heading/section` | `font/title2/strong` | Заголовок крупной секции |
| `type/heading/panel` | `font/title3/strong` | Панели, карточки, modal header |
| `type/heading/compact` | `font/headline/strong` | Компактные панели, тулбар таблицы, плотные заголовки |

Правила:
- На экране обычно один `type/heading/page`.
- Не используйте презентационные роли как shortcut для важности.
- Заголовок должен описывать содержимое, а не действие.

### Основной Текст

| Роль | Токен | Использование |
|---|---|---|
| `type/body/large` | `font/bodyLarge/base` | Описательный текст, онбординг, тело модального окна |
| `type/body/default` | `font/body/base` | Основной текст интерфейса |
| `type/body/small` | `font/bodySmall/base` | Плотный интерфейс, ячейка таблицы, компактная подсказка |
| `type/body/xsmall` | `font/bodyExtraSmall/base` | Микротекст, временные метки, плотные метаданные |

Правила:
- `type/body/default` является базовым текстом интерфейса.
- Для длинного чтения используйте `bodyLarge`, но не для таблиц.
- Для плотных таблиц используйте `bodySmall`, если контраст и читаемость сохраняются.

### Подписи

| Роль | Токен | Использование |
|---|---|---|
| `type/label/default` | `font/body/strong` | Подпись поля или контрола |
| `type/label/small` | `font/bodySmall/strong` | Компактная форма, фильтры таблицы, теги |
| `type/label/xsmall` | `font/bodyExtraSmall/strong` | Бейджи, маленькие подписи метаданных |

Правила:
- Подпись поля не заменяется placeholder.
- Подпись должна быть коротким существительным или понятной фразой.
- Текст кнопки описывает действие глаголом и относится к правилам контента, а не только к типографике.

### Вспомогательный Текст

| Роль | Токен | Использование |
|---|---|---|
| `type/caption/default` | `font/bodySmall/base` | Подсказка, вторичное пояснение |
| `type/caption/tiny` | `font/bodyExtraSmall/base` | Плотные временные метки, технические подсказки |
| `type/caption/micro` | `font/caption/base` | Только для крайне ограниченных технических мест |

Правила:
- `caption` не должен быть единственным источником важной информации.
- Не используйте `font/caption` для обычных helper texts.
- Минимальный практичный размер для важного текста интерфейса: 12px.

### Код

| Роль | Токен | Использование |
|---|---|---|
| `type/code/default` | `font/body/base` + моноширинный шрифт | Фрагменты кода, имена переменных |
| `type/code/small` | `font/bodySmall/base` + моноширинный шрифт | Встроенный код, имена токенов, идентификаторы |

Правила:
- Код использует моноширинный шрифт.
- Встроенный код не должен ломать line-height строки.
- Имена токенов и CSS-переменные пишутся без переносов внутри слова.

### Метрики

| Роль | Токен | Использование |
|---|---|---|
| `type/metric/large` | `font/title2/strong` | Значение метрики в дашборде |
| `type/metric/default` | `font/title3/strong` | Значение метрики в карточке |
| `type/metric/small` | `font/headline/strong` | Компактная метрика в таблице или панели |
| `type/metric/label` | `font/bodySmall/base` | Подпись метрики |

Правила:
- Числа выравнивайте таблично, если сравнение идет в колонке.
- Единица измерения визуально слабее значения: `bodySmall` или `bodyExtraSmall`.
- Тренд не передается только цветом.

## Связь С Размером Компонента

`sizes.md` задаёт высоту компонента, radius и типографику контрола. Типографические роли задают текстовую иерархию страницы.

| Размер компонента | Текст контрола | Использование |
|---|---|---|
| `small` | `font/bodySmall` 12 / 16 | Тулбар, плотные контролы таблиц, inline-действия |
| `medium` | `font/body` 14 / 20 | Формы, карточки, фильтры по умолчанию |
| `large` | `font/bodyLarge` 16 / 24 | Заметные формы, основные действия |
| `extraLarge` | `font/bodyExtraLarge` 18 / 28 | Сценарии с приоритетом touch, hero-сценарии, крупные CTA |

Правила:
- Не используйте размер компонента как иерархию страницы.
- `large` button не делает действие главным само по себе. Иерархию задают variant, placement и content.
- Label и helper text могут оставаться `font/body` 14 / 20 независимо от размера input, если это улучшает сканирование формы.

---

## Длина Строки И Ритм

| Контент | Рекомендация |
|---|---|
| Длинный body text | 60-80 символов в строке |
| Form helper text | До 1-2 строк |
| Table cell | По возможности одна строка, truncation с tooltip/details |
| Modal body | 45-70 символов в строке |
| Dense metadata | Короткие фразы, избегать длинных предложений |

Line-height должен оставаться из токена. Не уменьшайте line-height для экономии места, если текст может переноситься.

---

## Обрезка И Перенос

- Заголовки страниц и секций обычно переносятся.
- Подписи не должны обрезаться, если они критичны для понимания поля.
- Ячейки таблиц могут обрезаться, если есть способ увидеть полное значение.
- Текст кнопок не должен обрезаться в обычных сценариях.
- Имена токенов, идентификаторы и код могут иметь горизонтальный скролл или управляемый перенос.

---

## Числовой Текст

- Для сравнения чисел в таблицах используйте выравнивание по правому краю.
- Для колонок с числами используйте tabular numbers, если шрифт поддерживает.
- Единицы измерения отделяйте пробелом: `24 px`, `15 %`, если продуктовая локаль не требует иного.
- Статусы тренда сопровождайте иконкой или текстом, не только цветом.

Рекомендуемый CSS:

```css
font-variant-numeric: tabular-nums;
```

---

## Регистр

| Контекст | Правило |
|---|---|
| Текст кнопки | Глагол или глагольная фраза: `Сохранить`, `Удалить`, `Отправить` |
| Подпись поля | Короткое существительное: `Email`, `Название проекта` |
| Tabs | Название раздела: `Настройки`, `История` |
| Заголовок таблицы | Короткая подпись: `Статус`, `Дата`, `Владелец` |
| Текст ошибки | Объясняет проблему и способ исправления |

Не используйте ALL CAPS для обычных подписей. Если нужна служебная подпись в верхнем регистре, ограничьте её микротекстом и проверьте читаемость.

---

## Адаптивное Поведение

- Не масштабируйте font-size от ширины viewport.
- На мобильных переносите layout, а не уменьшайте текст ниже читаемого размера.
- Минимальный размер важного текста интерфейса: 12px.
- Презентационные роли на мобильных можно заменить на `heading/page` или `heading/section`.
- Длинные заголовки должны переноситься без перекрытия соседнего контента.

---

## Доступность

- Текст должен соответствовать `foundation/accessibility.md`.
- Контраст body, label и helper text: минимум `4.5:1`.
- Крупный текст: минимум `3:1`.
- Не передавайте смысл только italic, weight или цветом.
- Если текст является accessible name, он должен быть понятен без Tooltip.
- Placeholder не заменяет label.

---

## Что Делать И Чего Избегать

| Делать | Избегать |
|---|---|
| Используйте семантические роли для иерархии | Использовать размер компонента как заголовок страницы |
| Используйте `body` как базовый UI text | Делать весь интерфейс `bodySmall` ради плотности |
| Используйте `strong` для смыслового акцента | Делать все labels semibold без иерархии |
| Используйте моноширинный шрифт только для технических значений | Использовать моноширинный шрифт для декоративного стиля |
| Переносите layout на мобильных | Масштабировать шрифт от viewport |
| Проверяйте contrast для helper/caption | Считать маленький текст второстепенным для accessibility |

---

## Чеклист Токена

Перед добавлением нового text style:

- [ ] Нельзя решить задачу существующей семантической ролью.
- [ ] Известно, это базовый или семантический токен.
- [ ] Есть size, line-height, weight и letter-spacing.
- [ ] Есть сценарии использования и ограничения.
- [ ] Проверена связь с размерами компонентов.
- [ ] Проверен contrast для типичных поверхностей.
- [ ] Добавление не создает дубликат существующей роли.
