# Отступы и размеры

> Figma: TBD

Отступы и размеры задают ритм, плотность, размеры компонентов и правила компоновки интерфейсов SEDA UI. Документ покрывает шкалу отступов, семантические роли, режимы плотности, component sizing, правила grid/container и основные паттерны для форм, таблиц, дашбордов и сайдбаров.

---

## Цель

Отступы должны помогать:

- группировать связанные элементы;
- отделять разные смысловые блоки;
- сохранять плотность B2B и AI-интерфейсов;
- избегать произвольных значений margin и padding;
- строить адаптивные страницы без визуального шума.

Правило: используйте spacing tokens и CSS `gap` для отношений между элементами. Margin применяйте только для внешнего позиционирования блока в layout.

---

## Базовая Шкала Отступов

Текущая шкала из `tokens.json`:

| Токен | Значение | Использование |
|---|---:|---|
| `space/none` | 0px | Без отступа |
| `space/2xs` | 2px | Оптическая корректировка, микрозазоры |
| `space/xs` | 4px | Очень плотный внутренний gap |
| `space/s` | 6px | Внутренний gap компактных контролов |
| `space/m` | 8px | Малый gap по умолчанию |
| `space/l` | 10px | Плотный вертикальный ритм |
| `space/xl` | 12px | Отступ компактной группы |
| `space/2xl` | 16px | Отступ компонента или формы по умолчанию |
| `space/3xl` | 20px | Более свободный отступ компонента |
| `space/4xl` | 24px | Внутренний отступ секции |
| `space/5xl` | 32px | Крупный отступ секции |
| `space/6xl` | 40px | Padding desktop-контейнера |
| `space/7xl` | 56px | Крупный отступ секции страницы |

Примечание о текущем состоянии: старые документы упоминали только `4, 8, 12, 16, 24, 32`. Полная шкала выше должна считаться источником для новых specs.

## Семантические Роли Отступов

Primitive spacing описывает значения. Semantic spacing описывает назначение.

### Отступы Компонентов

| Роль | Токен | Использование |
|---|---|---|
| `space/component/2xs` | `space/2xs` | Оптический offset иконки, микрокоррекция |
| `space/component/xs` | `space/xs` | Плотный gap между иконкой и текстом |
| `space/component/s` | `space/s` | Внутренний gap компактного контрола |
| `space/component/m` | `space/m` | Внутренний gap контрола по умолчанию |
| `space/component/l` | `space/xl` | Увеличенный внутренний gap |
| `space/component/xl` | `space/2xl` | Крупные контролы, плотный padding карточки |

### Layout-Отступы

| Роль | Токен | Использование |
|---|---|---|
| `space/layout/xs` | `space/m` | Плотный stack внутри группы |
| `space/layout/s` | `space/xl` | Компактный gap между блоками |
| `space/layout/m` | `space/2xl` | Вертикальный ритм по умолчанию |
| `space/layout/l` | `space/4xl` | Между группами |
| `space/layout/xl` | `space/5xl` | Между крупными регионами |

### Отступы Секций

| Роль | Токен | Использование |
|---|---|---|
| `space/section/s` | `space/4xl` | Компактная секция страницы |
| `space/section/m` | `space/5xl` | Отступ секции по умолчанию |
| `space/section/l` | `space/6xl` | Крупный desktop-отступ секции |
| `space/section/xl` | `space/7xl` | Редкий крупный editorial/landing отступ |

> Семантические роли — рекомендуемые имена для документации. Если реализация ещё не экспортирует их как токены, мапьте их на primitive `space/*` значения.

## Режимы Плотности

Плотность управляет компактностью информации. Её нужно выбирать для продуктового контекста, а не для отдельного компонента в изоляции.

| Плотность | Использование | Типичный размер компонента | Типичные отступы |
|---|---|---|---|
| `compact` | Таблицы, тулбары, фильтры, админ-панели | `small` / `medium` | `space/m` до `space/2xl` |
| `regular` | Основные экраны приложения, формы, карточки | `medium` | `space/2xl` до `space/4xl` |
| `comfortable` | Экраны с приоритетом touch, онбординг, empty states | `large` / `extraLarge` | `space/4xl` до `space/6xl` |

Правила:

- Не смешивайте режимы плотности внутри одного ряда контролов.
- Таблицы и дашборды могут быть compact, но действия всё равно требуют доступной hit area.
- Comfortable density не означает, что везде нужны декоративные oversized cards.
- Плотность не должна уменьшать touch target ниже `44x44px`.

## Размеры компонентов

SEDA AI использует четыре базовых размера компонентов: `small`, `medium`, `large`, `extraLarge`. Размер задает высоту контрола, типографику внутри контрола, базовый radius и характер плотности. Это часть layout/density модели, поэтому таблица размеров живет здесь, а не в отдельном foundation-файле.

| Размер | Высота | Шрифт / строка | Radius | Типичный контекст |
|---|---:|---:|---:|---|
| `small` | 24px | 12px / 16px | 6px | Compact tables, toolbars, inline actions. |
| `medium` | 32px | 14px / 20px | 8px | Основной размер форм, фильтров и панелей. |
| `large` | 40px | 16px / 24px | 10px | Акцентные controls, просторные формы. |
| `extraLarge` | 48px | 18px / 28px | 12px | Touch-first сценарии, крупные CTA. |

Связанные foundation:

- текстовые роли и иерархия: `typography.md`;
- radius и border: `radius-border.md`;
- размеры иконок внутри controls: `iconography.md`;
- touch target и keyboard accessibility: `accessibility.md`.

Правила:

- Не используйте `large` или `extraLarge` как способ сделать заголовок. Для заголовков используйте роли из `typography.md`.
- В одном toolbar или compact row используйте один размер controls, если нет явной смысловой причины.
- `small` может быть визуально 24px, но интерактивная область должна оставаться доступной, особенно на touch.
- Размер компонента не заменяет layout density. Сначала выберите density (`compact`, `regular`, `comfortable`), затем размер controls.

## Брейкпоинты

SEDA использует следующие layout-брейкпоинты:

| Брейкпоинт | Ширина | Рекомендуемые колонки | Padding контейнера |
|---|---:|---:|---:|
| `xs` | 320px | 4 | 16px |
| `sm` | 640px | 4 | 24px |
| `md` | 768px | 8 | 32px |
| `lg` | 1024px | 12 | 40px |
| `xl` | 1280px | 12 | 40px |
| `2xl` | 1440px | 12 | 40px |

Правила:

- Меняйте layout на брейкпоинтах; не масштабируйте font-size от ширины viewport.
- На мобильных layout обычно ставит контент вертикально, если связь side-by-side не критична.
- Держите главные действия видимыми и близкими к сценарию, который они завершают.

## Контейнеры

Контейнер ограничивает ширину контента и задаёт горизонтальный padding.

| Размер | Max-width | Использование |
|---|---:|---|
| `sm` | 640px | Login, узкие формы |
| `md` | 768px | Документы, простые настройки |
| `lg` | 1024px | Формы со вспомогательным контентом |
| `xl` | 1280px | Дефолтный app layout и дашборды |
| `2xl` | 1440px | Широкие экраны данных |
| `full` | 100% | Full-width инструменты, data grids, canvas |

Правила:

- Используйте один container на секцию.
- Не вкладывайте контейнеры одного размера друг в друга.
- Используйте `full` для table-heavy или canvas-like tools, которым нужна ширина.
- Контент с большим количеством текста должен предпочитать `sm` или `md`, чтобы сохранить читаемую длину строки.

## Сетка

Используйте Grid для композиции страницы, а не для каждого маленького компонента.

| Тип | Использование |
|---|---|
| `12-column` | Layout страницы, регионы дашборда |
| `auto-fit` / `auto-fill` | Сетки карточек и адаптивные коллекции |
| `subgrid` / local grid | Формы и table-like выравнивание, если поддерживается |

Правила:

- Используйте `gap`, а не margins у дочерних элементов, для отношений в сетке.
- Grid gap должен приходить из spacing tokens.
- Избегайте одноразовой математики колонок, если проще работает flex/stack.
- Повторяющиеся карточки и панели выравнивайте по одним grid tracks.

Рекомендуемые gaps:

| Контекст | Gap |
|---|---|
| Плотный toolbar таблицы | `space/m` / 8px |
| Stack полей формы | `space/2xl` / 16px |
| Сетка карточек | `space/4xl` / 24px |
| Регионы дашборда | `space/4xl` до `space/5xl` / 24-32px |

## Вертикальный И Горизонтальный Layout

### Вертикальный Stack

Используйте Stack для вертикального ритма.

| Контекст | Gap |
|---|---|
| Подпись к контролу | `space/s` до `space/m` / 6-8px |
| Контрол к helper text | `space/xs` до `space/m` / 4-8px |
| Поля в форме | `space/2xl` / 16px |
| Группы полей | `space/4xl` / 24px |
| Секции страницы | `space/5xl` / 32px |

### Горизонтальный Inline

Используйте Inline для горизонтальных групп.

| Контекст | Gap |
|---|---|
| Иконка + подпись | `space/xs` до `space/s` / 4-6px |
| Button group | `space/m` / 8px |
| Контролы тулбара | `space/m` до `space/xl` / 8-12px |
| Действия формы | `space/xl` / 12px |
| Filter chips | `space/m` / 8px |

Правила:

- Держите inline-контролы выровненными по вертикали.
- Не смешивайте размеры компонентов в одной inline-группе без ясной причины.
- Если подписи переносятся, переключайтесь на vertical stack вместо сжатия текста.

## Layout-Паттерны

### Формы

Паттерн по умолчанию: vertical.

| Элемент | Отступ |
|---|---|
| Подпись к контролу | 6-8px |
| Контрол к helper/error | 4-8px |
| Поле к полю | 16px |
| Заголовок группы к первому полю | 12-16px |
| Группа полей к группе полей | 24-32px |
| Поля к submit actions | 24px |

Правила:

- Используйте vertical labels по умолчанию.
- Используйте horizontal labels только для коротких подписей и плотных enterprise-форм.
- Error summary находится над содержимым формы с `space/2xl` до первой группы полей.
- Действия выравниваются по контенту формы, а не по краю viewport.

### Таблицы

| Элемент | Отступ |
|---|---|
| Cell padding compact | 8px по горизонтали, 6-8px по вертикали |
| Cell padding regular | 12-16px по горизонтали, 10-12px по вертикали |
| Gap в toolbar | 8-12px |
| Toolbar до таблицы | 12-16px |
| Pagination до таблицы | 12-16px |

Правила:

- Плотные таблицы могут использовать compact spacing, но действия в строках всё равно требуют доступных hit areas.
- Избегайте card-like padding вокруг каждой строки.
- Используйте согласованный cell padding в header и body.
- Держите числовые колонки выровненными для быстрого сканирования.

### Дашборды

| Элемент | Отступ |
|---|---|
| Gap между регионами | 24-32px |
| Внутренний padding карточки/панели | 16-24px |
| Подпись метрики к значению | 4-8px |
| Filter bar к контенту | 16-24px |

Правила:

- Предпочитайте организованную плотность крупным декоративным карточкам.
- Группируйте фильтры рядом с контентом, на который они влияют.
- Используйте согласованный panel padding в рамках дашборда.
- Избегайте вложенных карточек внутри карточек.

### Master-detail Layout

| Регион | Рекомендация |
|---|---|
| Колонка списка | 280-360px |
| Колонка деталей | Гибкая ширина |
| Gap между колонками | 16-24px |
| Mobile | Разделять list и detail как отдельные views |

Правила:

- Сохраняйте состояние выбора видимым в списке.
- Контент деталей не должен смещаться, когда элементы списка меняют высоту.
- На маленьких экранах не показывайте обе панели одновременно без необходимости.

### Layout С Сайдбаром

| Регион | Рекомендация |
|---|---|
| Ширина compact-сайдбара | 64px |
| Ширина default-сайдбара | 240-280px |
| Padding контента | 24-40px по брейкпоинтам |
| Gap элемента навигации | 4-8px |

Правила:

- Отступы сайдбара и контента должны быть независимыми.
- Collapsed sidebar сохраняет hit areas доступными.
- Контент выравнивается по container/grid, а не по padding элементов сайдбара.

### Command/Search Overlay

| Элемент | Отступ |
|---|---|
| Overlay internal padding | 16-24px |
| Input to results | 8-12px |
| Result item padding | 8-12px |
| Group gap | 12-16px |

Правила:

- Приоритизируйте клавиатурное сканирование.
- Строки результатов должны иметь стабильную высоту.
- Избегайте избыточных вертикальных отступов в command palette.

## Safe Areas И Переполнение

- Учитывайте device safe areas для fixed headers, bottom bars и drawers.
- Используйте sticky headers только когда они помогают повторяющимся workflows.
- Scroll containers должны иметь ясные границы.
- Избегайте вложенных scroll regions, если workflow этого не требует.
- Popovers и menus должны перестраиваться у краёв viewport.

---

## Доступность

- Отступы не должны уменьшать touch target ниже `44x44px`.
- Focus ring не должен обрезаться через `overflow: hidden` или тесные контейнеры.
- Текст не должен перекрывать соседний контент при переносе.
- Плотные layouts всё равно требуют читаемого line-height из `typography.md`.
- Связанные labels, helper text и controls должны оставаться визуально сгруппированными.

---

## Что Делать И Чего Избегать

| Делать | Избегать |
|---|---|
| Использовать spacing tokens для gaps и padding | Использовать произвольные значения вроде 13px или 27px |
| Использовать `gap` для layout-отношений | Раздвигать соседние элементы случайными margins |
| Выбирать плотность по контексту | Смешивать compact и comfortable controls в одном ряду |
| Перестраивать layout на мобильных | Уменьшать текст ниже читаемого размера |
| Не обрезать focus rings | Помещать интерактивные controls в тесные overflow-hidden wrappers |
| Использовать full width для data grids | Втискивать широкие таблицы в узкие article containers |

---

## Чеклист Проверки Layout

- [ ] Значения отступов приходят из `space/*` tokens.
- [ ] Режим плотности понятен для экрана или секции.
- [ ] Выбор grid/container соответствует типу контента.
- [ ] Связанные элементы расположены ближе, чем несвязанные.
- [ ] Подписи формы, controls и helper text остаются сгруппированными.
- [ ] Таблицы используют согласованный cell padding.
- [ ] Mobile layout перестраивается или меняет структуру, а не уменьшает текст.
- [ ] Touch targets не меньше `44x44px`.
- [ ] Focus rings не обрезаются.
- [ ] Нет вложенных карточек внутри карточек.
