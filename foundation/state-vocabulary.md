# Словарь Состояний

Единый словарь состояний, используемый во всех компонентах SEDA UI.

Состояния разделены на категории:
- взаимодействие
- выбор
- данные
- валидация
- доступность
- асинхронные процессы

Этот документ фиксирует значение каждого состояния. Визуальное поведение состояний описано в `states.md`, а соответствие состояниям через токены — в `theming.md`.

---

## 1. Состояния Взаимодействия

### `hover`
Курсор наведен на интерактивный элемент.

- Временное состояние
- Даёт только визуальную обратную связь
- Не меняет смысл компонента

---

### `focus`
Элемент находится в фокусе клавиатуры.

- Используется для доступности
- Всегда сопровождается `focus/ring`
- Не означает выбранность или активность

---

### `active`
Момент нажатия (`mouse down` / `key press`).

- Временное состояние
- Длится только во время нажатия
- Не означает, что элемент выбран
- Не используется как синоним `selected`

---

## 2. Состояния Выбора

### `selected`
Элемент выбран из набора взаимоисключающих вариантов.

- Постоянное состояние
- Используется для одиночного выбора
- Примеры: Radio, Segmented Control, Button Group в режиме переключения

---

### `checked`
Бинарный выбор (`true` / `false`).

- Постоянное состояние
- Используется только для компонентов, похожих на Checkbox
- Не заменяет `selected`

---

### `indeterminate`
Частично выбранное состояние.

- Используется только для родительских checkbox
- Показывает, что выбрана только часть дочерних элементов
- Не равен `checked`

---

### `on` / `off`
Включено / выключено.

- Используется только для Toggle / Switch
- Описывает состояние настройки
- Не заменяет `selected` или `checked`

---

## 3. Состояния Данных

### `filled`
У компонента есть значение (`value ≠ empty`).

- Не означает валидность
- Показывает только факт наличия данных
- Примеры: Text Field, Text Area, Select, Date Picker, Time Picker, Verification Code

---

### `open`
Компонент раскрыт.

- Относится к overlay, dropdown или popup-части компонента
- Не связан напрямую с валидностью
- Примеры: Select, Date Picker, Time Picker, Dropdown, Popover

---

## 4. Состояния Валидации

### `error`
Данные невалидны или операция завершилась ошибкой.

- Показывается после взаимодействия или попытки отправки
- Должен сопровождаться текстом ошибки или понятным объяснением

---

### `warning`
Обнаружена потенциальная проблема.

- Не всегда блокирует действие
- Используется как предупреждение, а не как фатальная ошибка

---

### `success`
Данные валидны или операция завершилась успешно.

- Используется как положительная обратная связь
- Не заменяет содержательную подпись, если она нужна

---

## 5. Состояния Доступности

### `disabled`
Элемент недоступен для взаимодействия.

- Блокирует `hover`, `focus` и `active`
- Исключает или ограничивает взаимодействие в зависимости от платформы и нативного поведения
- Не объясняет причину недоступности сам по себе

---

### `read-only`
Элемент доступен для чтения, но не редактируется.

- Значение остаётся видимым
- Значение можно копировать
- Обычно остаётся в tab-order
- Используется только там, где есть редактируемое значение

---

## 6. Асинхронные Состояния

### `loading`
Операция в процессе.

- Временное, но функционально значимое состояние
- Обычно блокирует повторное взаимодействие
- Показывает spinner, progress indicator или другую обратную связь ожидания

---

## 7. Состояния Конкретных Компонентов

Эти состояния допустимы только внутри конкретных компонентов и не считаются глобальными терминами системы:

- `drag-over`
- `uploading`
- `visited`
- `expanded`
- `collapsed`
- `current`
- `skeleton`

Их можно использовать в документации конкретного компонента, но не как основу общего словаря состояний.

---

## Правила

### `active` ≠ `selected`
- `active` — только состояние нажатия
- `selected` — устойчивое состояние выбора

### `checked` ≠ `selected`
- `checked` — бинарный выбор
- `selected` — выбор одного варианта из набора

### Toggle использует `on/off`
- Не использовать `active` как термин для состояния Toggle
- Внутренняя реализация может быть построена на checkbox, но публичная семантика остаётся `on/off`

### `filled` ≠ `valid`
- Наличие значения не означает корректность значения

### `disabled` ≠ `read-only`
- `disabled` полностью ограничивает взаимодействие
- `read-only` оставляет доступ к значению без возможности редактирования

---

## Итоговая модель

| Категория | Состояния |
|---|---|
| Взаимодействие | `hover`, `focus`, `active` |
| Выбор | `selected`, `checked`, `indeterminate`, `on/off` |
| Данные | `filled`, `open` |
| Валидация | `error`, `warning`, `success` |
| Доступность | `disabled`, `read-only` |
| Асинхронные процессы | `loading` |
