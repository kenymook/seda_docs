# Модель Взаимодействия

Общие правила взаимодействия компонентов в SEDA UI.

Этот документ фиксирует базовые паттерны поведения: фокус, активацию, выбор, открытие и закрытие составных компонентов. Семантика состояний описана в `state-vocabulary.md`, а визуальное отображение — в `states.md`.

---

## 1. Принципы

- **Сначала нативное поведение.** Если нативный HTML-элемент уже даёт корректное поведение, используйте его как основу.
- **Одно действие — один ожидаемый результат.** Click, Enter и Space не должны запускать разные сценарии без явной причины.
- **Паритет клавиатуры.** Всё, что доступно мышью, должно быть доступно с клавиатуры.
- **Предсказуемый фокус.** Пользователь всегда понимает, где находится фокус и куда он перейдёт дальше.

---

## 2. Модель Фокуса

### Навигация Tab
- `Tab` переводит фокус к следующему интерактивному элементу
- `Shift+Tab` переводит фокус к предыдущему интерактивному элементу
- `disabled`-элементы не участвуют в tab-order
- `read-only`-поля обычно остаются в tab-order

### Видимость Фокуса
- Focus ring обязателен для всех интерактивных компонентов
- Индикатор фокуса не должен зависеть только от цвета
- Фокус должен быть видим на любом фоне

### Составные Компоненты
Для составных компонентов используются два базовых паттерна:

1. **Фокус остаётся на триггере**
   - Select
   - Date Picker trigger
   - Popover trigger

2. **Фокус переходит внутрь составного виджета**
   - Radio Group
   - Segmented Control
   - OTP / Verification Code
   - Calendar grid

---

## 3. Модель Активации

### Стандартная Активация
- `Click` → активация
- `Enter` → активация
- `Space` → активация для button-like контролов

### Нажатие И Выбор
- `active` = момент нажатия
- `selected` / `checked` = устойчивый результат выбора

Не использовать `active` как термин для выбранного состояния.

---

## 4. Паттерны Выбора

### Одиночный Выбор
Используется там, где из набора можно выбрать только один вариант.

Примеры:
- Radio Group
- Segmented Control
- Button Group в режиме переключения
- Select в режиме одиночного выбора

Правила:
- в один момент времени выбран только один вариант
- навигация между вариантами может идти стрелками
- устойчивое состояние обозначается как `selected`

### Множественный Выбор
Используется там, где можно выбрать несколько вариантов.

Примеры:
- Checkbox Group
- Select в режиме множественного выбора

Правила:
- каждый элемент независим
- устойчивое состояние обозначается как `checked`
- родительский checkbox может использовать `indeterminate`

### Toggle
Используется для мгновенного включения / выключения настройки.

Примеры:
- Toggle / Switch

Правила:
- публичное состояние обозначается как `on` / `off`
- изменение применяется сразу, без submit

---

## 5. Модель Открытия И Закрытия

### Открытие
Составной компонент открывается по одному из триггеров:
- `Click`
- `Enter`
- `Space`
- иногда — по `ArrowDown` для компонентов, похожих на Select

### Закрытие
Составной компонент закрывается по одному из событий:
- `Escape`
- клик вне компонента
- выбор опции, если сценарий одношаговый
- явное действие пользователя (`Close`, `Cancel`)

### Фокус После Закрытия
После закрытия фокус возвращается:
- на триггер компонента
- либо на логически следующий элемент, если сценарий завершён

---

## 6. Клавиатурные Паттерны По Семействам Компонентов

### Компоненты, Похожие На Кнопку
Компоненты:
- Button
- Icon Button
- Toggle

Клавиши:
- `Tab` / `Shift+Tab`
- `Enter`
- `Space`

### Компоненты, Похожие На Поле Ввода
Компоненты:
- Text Field
- Text Area
- Date Picker input
- Time Picker input

Клавиши:
- `Tab` / `Shift+Tab`
- текстовый ввод
- клавиши, специфичные для компонента, при необходимости

### Выбор В Списке
Компоненты:
- Radio Group
- Segmented Control
- Select options

Клавиши:
- `Tab` / `Shift+Tab`
- `Arrow keys`
- `Enter` / `Space`
- `Escape` для закрываемых контейнеров

### Контролы Диапазона
Компоненты:
- Slider

Клавиши:
- `Tab`
- `Arrow keys`
- `Home`
- `End`

---

## 7. Поведение Валидации

- `error`, `warning` и `success` не должны появляться до взаимодействия без явной причины
- пустое поле по умолчанию не считается ошибкой
- `filled` не означает `valid`
- при `error` компонент остаётся редактируемым, если не указан иной сценарий

---

## 8. Disabled И Read-only

### `disabled`
- не принимает ввод
- не открывается
- не активируется
- обычно исключается из tab-order

### `read-only`
- не редактируется
- значение остаётся видимым
- значение можно скопировать
- обычно остаётся в tab-order

---

## 9. Асинхронная Обратная Связь

### `loading`
Используется, когда действие уже запущено и пользователь должен получить обратную связь ожидания.

Правила:
- блокирует повторный запуск того же действия
- не должен приводить к скачкам layout
- должен быть визуально заметен

### Асинхронные Сценарии С Прогрессом
Для сценариев с известным прогрессом используйте progress pattern вместо неопределённого spinner.

Примеры:
- File Upload
- long-running import / export

---

## 10. Touch Targets

- Минимальный touch target: `44×44px`
- Для маленьких визуальных контролов зона касания расширяется невидимым padding
- Hover не считается обязательным состоянием для touch-only устройств
